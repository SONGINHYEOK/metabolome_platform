# 시연 영상 시나리오

## 영상 개요
- **목적**: 발표평가용 프로토타입 데모
- **예상 시간**: 4~5분
- **URL**: http://127.0.0.1:6321/
- **핵심 메시지**: 대사체 데이터의 생산·표준화·활용 전주기를 디지털화하는 2-Track 통합 플랫폼 + Groq AI 기반 실시간 해석

---

## Scene 1. 도입 — 왜 이 플랫폼이 필요한가 (30초)

**화면**: `/` (랜딩페이지 전체)

**나레이션**:
> "현재 특용작물 대사체 데이터는 연구기관별로 분산되어 있고, 분석 방법과 품질 기준이 통일되지 않아 데이터 재활용과 비교 분석이 어렵습니다. 본 플랫폼은 **80종 대사체 DB 구축**을 목표로, 데이터 생산부터 표준화, 활용까지 전주기를 지원하는 **2-Track 통합 디지털 플랫폼**입니다."

**동작**:
1. 랜딩페이지 전체 화면 보여주기
2. 상단 수치 카드 3개에 마우스 천천히 이동 (80종 → 50종 → 2-Track)
3. 하단 2-Track 선택 카드에 마우스 호버 — 카드 상승 애니메이션 확인
4. **RESEARCH PORTAL** 클릭

---

## Scene 2. 연구자 포털 메인 (20초)

**화면**: `/research/`

**나레이션**:
> "연구자 포털에서는 Compound Catalog, 원산지 판별, 표준 데이터 패키지, QC 검증 등 전문 분석 기능을 제공합니다."

**동작**:
1. 네이비 헤더와 브레드크럼 확인
2. Quick Stats (등록 성분, 분석 작물) 강조
3. **Compound Catalog** 카드(ACTIVE 배지)를 바로 클릭
   - *TIP: "개발 예정" 카드는 빠르게 스쳐 지나가고, 구현된 Compound Catalog에 집중*

## Scene 3. Compound Catalog (1분 30초) — 핵심 시연

**화면**: `/research/catalog/`

**나레이션 (진입 시)**:
> "Compound Catalog은 3-컬럼 레이아웃으로 설계되었습니다. 좌측 필터 패널에서 작목, 부위, 산지, 연도, QC 상태를 **조건 기반으로 탐색**할 수 있습니다."

### 동작 1 — 전체 레이아웃 (15초)
1. 3-컬럼 구조 전체 보여주기 (필터 | 목록 | 상세)
2. 자동 선택된 첫 번째 성분의 상세패널 확인

### 동작 2 — 필터링 시연 (20초)
1. 좌측 작목 필터에서 **"인삼"** 선택 → 목록이 인삼 성분만 필터링
2. 상단 "총 16건 중 6개 표시" 등 건수 변경 확인
3. **필터 초기화** 클릭하여 전체 목록 복원

### 동작 3 — 성분 선택 및 메타 정보 (20초)
1. 목록에서 **Ginsenoside Rg1** 클릭 → 우측 상세패널 동적 갱신
2. **MSI L1** 배지 강조 — "MSI 레벨 체계에 따라 동정 신뢰도를 표시합니다"
3. IN-HOUSE 배지, Score 96, Similarity 0.94 확인
4. 메타 정보 그리드 (작목, 부위, 산지, 연도) 확인

### 동작 4 — 상세 분석 패널 (25초)
1. 활용모델 라디오 버튼 (원산지 판별 / 성분 패턴 분석)
2. **TOP 3 CONTRIBUTION** 바 차트 (Saponin 65%, Sugar 25%, Aglycone 10%)
3. **QUICK ANALYSIS** — 원산지 판별 결과 HIGH, SCORE 96, SIMILARITY 0.94

**나레이션**:
> "각 성분별 Annotation Level과 Library Match 정보, 종합 Score를 **근거 기반**으로 제공합니다. MSI 레벨 체계에 따라 동정 신뢰도를 투명하게 표시하여, **데이터 표준화와 품질 관리**를 플랫폼 차원에서 지원합니다."

### 동작 5 — AI 실시간 해석 (20초)

**나레이션**:
> "성분을 선택하면 **Groq LLM 기반 AI 해석**이 실시간으로 생성됩니다. 동정 신뢰도 평가, 생리활성 의의, 원산지 기여도를 자동으로 분석하여 연구자의 의사결정을 지원합니다."

1. 상세패널 스크롤 → **AI 해석** 섹션(GROQ LLM ANALYSIS) 강조
2. 신뢰도 평가 / 생리활성 의의 / 원산지 기여도 / 한줄요약 항목 보여주기
3. 다른 성분 (**Decursin** 등) 클릭 → AI 해석이 새로 로딩되는 것 확인
4. 하단 스크롤 → 전문가용 근거 분석 CONCEPT IMAGE (A/B/C 카드)

## Scene 4. 대국민 포털 전환 (15초)

**화면**: `/public/` (브라우저 뒤로가기 → 랜딩 → PUBLIC PORTAL 클릭)

**나레이션**:
> "이제 대국민 서비스인 Public Portal로 이동합니다. 소비자, 농업인, 지자체를 대상으로 **쉬운 언어**의 성분 정보와 비교 분석을 제공합니다."

**동작**:
1. 대상별 카드 (소비자/농업인/지자체) 빠르게 보여주기
2. **작목 상세 대시보드** 카드 바로 클릭
   - *TIP: "개발 예정" 카드는 언급하지 않고 구현된 대시보드에 집중*

## Scene 5. 작목 상세 대시보드 (1분 30초) — 핵심 시연

**화면**: `/public/dashboard/`

### 동작 1 — 공공 API 연계 환경 지표 (20초)

**나레이션**:
> "작목 상세 대시보드에서는 **공공 API 연계** 지역·환경 지표를 확인할 수 있습니다. 연평균 기온, 강수량, 토양질 지수를 API 기반으로 제공하여, 재배 환경과 대사체 성분의 관계를 탐색할 수 있습니다."

1. 지역·환경 지표 카드 3개 강조 (12.8°C / 1150mm / A등급)
2. **"API 기반"** 배지를 마우스로 가리키며 강조

### 동작 2 — 성분 분포 및 비교 차트 (20초)

**나레이션**:
> "성분 분포 요약 차트에서 인삼의 주요 성분 Score를 한눈에 확인할 수 있으며, 비교 결과 분석 차트에서 **인삼과 황기의 성분별 차이**를 시각적으로 비교합니다."

1. 성분 분포 바 차트 보여주기
2. 비교 결과 그룹 바 차트 (인삼 파란색 vs 황기 주황색) 강조

### 동작 3 — 비교 대상 변경 (20초)
1. 좌측 사이드바에서 TARGET B를 **"당귀"**로 변경
2. **"비교 분석 적용"** 버튼 클릭
3. 차트가 인삼 vs 당귀로 동적 갱신되는 것 확인
4. 환경 지표도 해당 지역 데이터로 변경 확인

### 동작 4 — AI 동적 해설 및 대국민 산출물 (30초)

**나레이션**:
> "우측 패널에서는 **Groq LLM이 실시간으로 생성한** 비교 해설을 제공합니다. 하드코딩된 텍스트가 아니라, 선택한 작물 조합에 따라 **동적으로 AI가 분석 결과를 생성**합니다. 정책·산업·농업인별 활용 인사이트도 AI가 맞춤 생성하며, 불확실성 안내를 통해 데이터의 **신뢰성 범위도 투명하게** 표시합니다."

1. 우측 패널 상단 **"GROQ LLM 기반 동적 생성"** 배지 강조
2. AI 해설 콘텐츠(비교 요약) 읽기
3. **환경 영향 분석** 섹션 보여주기
4. 활용 시나리오 3개 (정책/산업/농업인) — AI 맞춤 생성된 내용 확인
5. 불확실성 안내 박스 강조
6. 대국민 산출물 미리보기 카드 보여주기
7. 하단 OUTPUT DOWNLOADS 버튼들 (요약카드, 비교카드, PDF 리포트)

**나레이션**:
> "이처럼 대국민 포털에서는 **표준 데이터 패키지** 기반의 산출물을 요약카드, 비교카드, PDF 리포트 형태로 다운로드할 수 있도록 설계하였습니다."

## Scene 6. AI 어시스턴트 챗봇 (30초)

**화면**: 대시보드 또는 카탈로그 페이지에서 챗봇 열기

**나레이션**:
> "플랫폼 전체에서 사용할 수 있는 **AI 어시스턴트 챗봇**도 제공합니다. Groq LLM 기반으로 대사체·작물 관련 질의에 실시간 응답하며, 현재 보고 있는 페이지의 컨텍스트를 자동으로 인식하여 맥락에 맞는 답변을 제공합니다."

**동작**:
1. 우하단 **파란색 챗봇 버튼** 클릭 → 채팅 패널 열기
2. "인삼과 황기의 주요 성분 차이는?" 입력 → 전송
3. AI 답변이 실시간으로 표시되는 것 확인
4. 추가 질문 "Ginsenoside Rg1의 생리활성은?" 입력 → 대화 이어지기 확인
5. 챗봇 닫기

## 촬영 팁

- **해상도**: 1920x1080 (Full HD) 권장
- **브라우저**: Chrome, 전체화면 또는 탭만 녹화
- **마우스**: 천천히 이동, 클릭 포인트에서 1초 정도 멈춤
- **전환**: 페이지 이동 시 브레드크럼 활용하면 자연스러움
- **녹화 도구**: OBS Studio 또는 macOS 기본 화면 녹화 (Cmd+Shift+5)
- **"개발 예정" 처리**: 구현된 기능(Compound Catalog, 작목 대시보드)에 집중하고, 미구현 카드는 빠르게 지나가기
- **핵심 키워드**: MSI 레벨, 공공 API 연계, 표준 데이터 패키지, 근거 기반, 2-Track 구조, **Groq LLM 기반 AI 해석**, **실시간 동적 생성**, AI 챗봇 — 나레이션에 자연스럽게 포함
- **AI 시연 팁**: AI 응답은 실시간 생성이므로 2~5초 로딩 시간이 있음. 로딩 애니메이션이 보이는 동안 나레이션으로 설명하면 자연스러움
